Հերթ (Queue)

Հերթը (Queue) գծային տվյալների կառուցվածք է, որն աշխատում է FIFO (First In, First Out) սկզբունքով։ Սա նշանակում է, որ առաջինը հերթում ավելացված («enqueue» եղած) տարրն է առա՜ջինը դուրս բերվում («dequeue» արվում)։ Կարելի է համեմատել խանութի հաճախորդների հերթի հետ․ ով առաջինն է կանգնել, նրան էլ սպասարկում են առաջինը։

Հիմնական սկզբունք: Հերթում առաջին ավելացված տարրն առաջինն է հեռացվում (FIFO)։

Օգտագործման օրինակներ

  • Տպիչի հերթ․ տպվելու հանձնված փաստաթղթերը մշակվում են ավելացման կարգով։
  • Օպերատորական սպասարկում․ բանկի կամ սպասարկման կենտրոնի հաճախորդները սպասվում են հերթ իրենց հերթականությամբ։
  • ՕՀՊ պլանավորիչ (CPU scheduler)․ որոշ պլանավորման ալգորիթմներում գործընթացները սպասում են ուղիղ հերթում։
  • Նախարարությունների դիմումների մշակություն․ դիմումները հերթագրվում են և մշակվում առաջինից սկսած։
  • Տվյալների հոսքերի բուֆերավորում․ ուսանողին ընթերցվող կամ ուղարկվող փաթեթները պահվում են հերթում՝ ուղարկվելու կամ մշակվելու համար։

Հիմնական գործողություններ

  • Enqueue («ավելացնել»)․ Նոր տարրի ավելացում հերթի հետևից։
  • Dequeue («հեռացնել»)․ Առաջին տարրի հեռացում հերթի սկզբից։
  • Front / Peek («նայել առաջին տարրը»)․ Վերադարձնում է առաջին տարրը՝ առանց այն հեռացնելու։
  • Size («չափ»)․ Վերադարձնում է հերթում գտնվող տարրերի քանակը։
  • isEmpty («դատարկ է»)․ Ստուգում է, արդյոք հերթը դատարկ է։

Կոդային օրինակ

function InitializeQueue():
    items = []     
    return items

function Enqueue(queue, item):
    append item to queue
    return queue

Թվային օրինակ

Սկսում ենք դատարկ հերթից

[]

Տարրերի ավելացում (Enqueue)

Ենթադրենք, հերթով ավելացնում ենք տարրերը՝ 4, 7 և 2.

Enqueue(4) → [4]
Enqueue(7) → [4, 7]
Enqueue(2) → [4, 7, 2]   (Հերթի սկզբում 4-ն է, վերջում — 2)

Տարրի հեռացում (Dequeue)

Երբ հեռացնում ենք տարր, առաջինը դուրս է գալիս ամենա֊առաջինը ավելացված տարրը (4)․

Dequeue() → 4  (Հերթը դառնում է [7, 2])
Dequeue() → 7  (Հերթը դառնում է [2])
Dequeue() → 2  (Հերթը դառնում է [])
Dequeue() → "Queue is empty"

Ժամանակային բարդություն

  • Enqueue («ավելացնել»)․ O(1) – տարրի ավելացումը կատարվում է հաստատուն ժամանակում։
  • Dequeue («հեռացնել»)․ O(1) – առաջին տարրի հեռացումը կատարվում է հաստատուն ժամանակում։
  • Front («նայել»)․ O(1) – առաջին տարրին հասանելիությունն իրականացվում է հաստատուն ժամանակում։
  • isEmpty («դատարկ է»)․ O(1) – դատարկ լինելու ստուգումը հաստատուն ժամանակում։
  • Size («չափ»)․ O(1) – տարրերի քանակի վերադարձը հաստատուն ժամանակում, եթե այն պահվում է հաշվված։

Ինտերակտիվ Վիզուալիզացիա

Մուտքագրեք արժեք և ավելացրեք այն հերթին (Enqueue) կամ հանեք առաջին տարրը (Dequeue)։ Դիտեք, թե ինչպես է աշխատում FIFO սկզբունքը․
